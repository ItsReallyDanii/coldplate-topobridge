## 4. Critical Structural Finding — G5 Criterion Reformulation Required

**Bridge results from running both controls at z=25:**

| Geometry | theta_std (rad) | n_solid | n_active | G5/G6 condition |
|---|---|---|---|---|
| uniform_channel_ctrl | **1.5107** | 0 | 2489 | ❌ Not < 0.05 |
| single_obstruction_ctrl | **1.7948** | 100 | 2400 | (+0.284 vs uniform) |
| cand_01 diamond (reference) | 1.8081 | 1094 | 1406 | — |

**Root cause of G5 failure:**

The pressure-Poisson + Darcy solver produces `vx=vy=0` (floating-point zero ~ 1e-9 m/s)
for the all-fluid uniform channel. The bridge computes `theta_bridge = atan2(vy, vx)`.
When both vy and vx are at machine epsilon, `atan2` returns arbitrary angles depending
on the sign bit of the floating-point zero (e.g., `atan2(-0.0, +0.0)` ≠ `atan2(+0.0, -0.0)`).

This produces `theta_std ≈ 1.51 rad` even though the physics implies zero transverse flow.
The bridge's current low-signal threshold (0.01 × max_magnitude of the vx/vy field) does
NOT catch this because: when the entire field has |vx|,|vy| ~ 1e-9, max_magnitude is also
~1e-9, and threshold = 0.01 × 1e-9 = 1e-11, which almost all pixels exceed.

**This is documented as U-014 in `docs/UNCERTAINTIES.md`.**

**G5 criterion must be reformulated:**

The criterion `theta_std < 0.05` is structurally incompatible with the Darcy solver's
zero-transverse-velocity representation. The correct G5 criterion must instead assert
something that CAN be measured from the actual field:

**Reformulated G5 criterion:**
```
G5_pass = (n_solid == 0) AND (vx_mag_max / vz_mean < 1e-5)
          AND (n_low_signal_vx / n_total > 0.99)
```
Or equivalently in the bridge, a separate "transverse magnitude" check:
```
transverse_max_ratio = max(|vx|, |vy|) / mean(|vz|) < 1e-5  → uniform flow confirmed
```

**G6 criterion (updated):** `theta_std(obstruction) > theta_std(uniform) + 0.1 rad`
which is satisfied: 1.7948 > 1.5107 + 0.1 = 1.6107 → 1.7948 > 1.6107 ✅

G6 is satisfied by the obstruction geometry IF G5 is redefined.
