# Real Input VALIDATED Tier — Upstream Requirements
# Date: 2026-03-10

This document defines the exact upstream data requirements holding the bridge at the `IMPLEMENTED` tier. It specifies what must be generated by `coldplate-design-engine` before the `VALIDATED` tier can be reached for the real-artifact path.

---

## 1. Current Status Snapshot

**What is IMPLEMENTED:**
- Stage 4 artifact ingestion (`velocity_field.npz`, `provenance.json`).
- Synthetic control testing (uniform channel = low variance, swirl block = high variance).
- Parsing, JSON-schema validation, and deterministic signature generation on real data.
- Comparison test harness (`tests/test_real_comparison.py`) capable of parsing real candidate runs.
- Successful run on diamond TPMS candidates 01 and 02.

**What is VALIDATED:**
- The bridge pipeline code itself runs without crashing on real data.
- Outputs for equal inputs are strictly deterministic (by bitwise JSON hash).
- Outputs for unequal inputs are correctly differentiated.
- Schema conformity is 100%.

**What is NOT PROVEN:**
- Any physical or scientific interpretation of bridge metrics.
- That the bridge's response to geometry variation is monotonic or properly scaled to TopoStream goals.
- That Darcy field angle derivatives act as a stable structural signal.
- The `VALIDATED` tier is NOT reached because the bridge has not processed a known "ground truth" negative and positive control from the upstream parent repository.

## 3. Exact Missing Upstream Artifacts

To clear G5 and G6, two new artifacts must be generated by `coldplate-design-engine` and placed in the results tree.

### 1. The G5 Artifact (Negative Control)
- **Repo path:** `results/stage4_sim_full/baseline_uniform_channel...` (or similar).
- **Required contents:**
  - `velocity_field.npz`
  - `provenance.json`
- **Purpose:** A straight, uniform channel with zero flow obstructions. This flow field must be uniform parallel to the dominant axis.
- **Expected bridge behavior:** `theta_std` should be near 0.0.

### 2. The G6 Artifact (Positive Control)
- **Repo path:** `results/stage4_sim_full/baseline_single_obstruction...` (or similar).
- **Required contents:**
  - `velocity_field.npz`
  - `provenance.json`
- **Purpose:** A uniform channel containing exactly one simple flow obstruction (e.g., a single circle or square).
- **Expected bridge behavior:** `theta_std` should be distinctly > 0, and spatial active records should spike around the obstruction.

---

## 4. Exact Artifact Requirements

Both missing artifacts must meet these constraints to be a valid comparison to the existing `diamond_2d` geometries:

1. **Solver consistency:**
   - They must be simulated using the exact same Stage 4 pressure-Poisson + Darcy solver.
   - They must use the identical grid resolution type (`50x50x50` for full).
   - They must use identical pressure drop boundary conditions.

2. **File names:**
   - Must contain `velocity_field.npz` (keys: `vx`, `vy`, `vz`, float64).
   - Must contain `provenance.json`.

3. **Provenance fields:**
   - `simulation.grid.porosity` must exist.
   - `git_sha` must exist (ideally the same code commit used for the diamond candidates).
   - `stage3_source.provenance.stage2_source.family` must exist (e.g., set to `"channel_uniform"`).

4. **Comparability conditions:**
   - The same slice index (e.g., `z=25` or `z=10`) must be valid for the control.

## 5. Acceptance Criteria for Later Bridge Rerun

Before anyone runs the validation step to clear `VALIDATED` tier, the following must be true:

- [ ] Upstream `coldplate-design-engine` has successfully emitted the two requested geometries.
- [ ] No `topobridge` code changes are required to ingest the new files beyond updating the test paths.
- [ ] `tests/test_real_comparison.py` is updated to compare the negative control to the positive control and then against `candidate_01`.
- [ ] **G5 condition:** The negative control produces `theta_std < 0.05` rad.
- [ ] **G6 condition:** The positive control produces `theta_std` clearly > negative control `theta_std` by at least 1 order of magnitude.

---

## 6. Non-Goals

- **NO TopoStream claims:** Proving that the positive control has variance does not mean the variance is equivalent to a TopoStream vortex token.
- **NO geometry ranking claims:** A lower `theta_std` does not mean the design is better, or has less pressure drop, or transfers more heat.
- **NO physical interpretation claims:** The angle distribution relates to Darcy flow approximation, not a Navier-Stokes streamline.
